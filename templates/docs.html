Overview

gitraf is a lightweight, self-hosted git server ecosystem. It provides simple, fast git hosting that's free from corporate control.

Architecture

+-------------------+     +-------------------+     +-------------------+
|   gitraf CLI      |---->|   SSH/HTTPS       |---->|   ogit server     |
|   (client)        |     |                   |     |   (git hosting)   |
+-------------------+     +-------------------+     +-------------------+
                                                            |
                          +---------------+-----------------+---------------+
                          |               |                                 |
                          v               v                                 v
                  +---------------+ +---------------+               +---------------+
                  | gitraf-server | | gitraf-pages  |               |   Git LFS     |
                  | (web UI)      | | (static sites)|               |   (S3 storage)|
                  +---------------+ +---------------+               +---------------+

Components

Component Description
gitraf CLI tool for managing repositories from your local machine
gitraf-server Web interface for browsing repositories (this page)
gitraf-pages Static site hosting from git repositories
gitraf-deploy Interactive deployment script for server setup

Access Model

Location Private Repos Public Repos
On Tailnet SSH read/write SSH read/write
External Not accessible HTTPS read-only

Installation

Quick Start (Server)

Deploy on a fresh Ubuntu server with the interactive script:

curl -sSL https://raw.githubusercontent.com/RafayelGardishyan/gitraf-deploy/main/deploy.sh | sudo bash

The script will guide you through:

  • Choosing components (core only, web UI, or full stack with pages)
  • Selecting access model (tailnet-only, public, or hybrid)
  • Configuring your domain and SSL certificates

CLI Installation

Install the gitraf CLI on your local machine:

git clone https://git.rafayel.dev/gitraf.git
cd gitraf
./install.sh

Or manually:

git clone https://git.rafayel.dev/gitraf.git
cd gitraf
go build -o gitraf .
sudo mv gitraf /usr/local/bin/

Configuration

Initialize gitraf with your server URLs:

gitraf config init https://git.yourdomain.com yourserver.tail12345.ts.net

Requirements

  • Server: Ubuntu 20.04+ or Debian 11+
  • CLI: Go 1.21+ for building
  • Tailscale: Required for private access
  • Domain: Required for public/hybrid access

CLI Reference

The gitraf CLI is your main interface for managing repositories on your self-hosted git server.

Installation

# Clone and build
git clone https://git.rafayel.dev/gitraf.git
cd gitraf
go build -o gitraf .
sudo mv gitraf /usr/local/bin/

# Or use the install script
./install.sh

Configuration

Before using gitraf, configure it with your server URLs:

# Initialize configuration (interactive)
gitraf config init

# Initialize with specific URLs
gitraf config init https://git.yourdomain.com yourserver.tail12345.ts.net

# Show current configuration
gitraf config show

# Set individual values
gitraf config set public_url https://git.yourdomain.com
gitraf config set tailnet_url yourserver.tail12345.ts.net
gitraf config set ssh_user git

Configuration is stored in ~/.config/gitraf/config.json:

{
  "public_url": "git.yourdomain.com",
  "tailnet_url": "yourserver.tail12345.ts.net",
  "ssh_user": "git"
}

Repository Management

Listing Repositories

# List all repositories (public + private if on tailnet)
gitraf list

# Output shows name, visibility, and description
# Example:
#   myproject     [public]   A cool project
#   private-app   [private]  Internal tooling

Repository Information

# Get detailed info about a repository
gitraf info myrepo

# Shows: clone URLs, visibility, size, last commit

Cloning Repositories

# Clone a repository (auto-selects best protocol)
gitraf clone myrepo

# On tailnet: uses SSH (git@server:myrepo.git)
# Off tailnet: uses HTTPS (https://git.domain.com/myrepo.git)

# Clone to specific directory
gitraf clone myrepo ./my-local-folder

Creating Repositories

# Create a new repository (requires tailnet)
gitraf create myrepo

# Create with description
gitraf create myrepo --description "My awesome project"

# Create as private (default) or public
gitraf create myrepo --public

Deleting Repositories

# Delete a repository (requires tailnet, prompts for confirmation)
gitraf delete myrepo

# Skip confirmation (dangerous!)
gitraf delete myrepo --force

Visibility Control

# Make repository publicly accessible via HTTPS
gitraf public myrepo

# Make repository private (tailnet-only access)
gitraf private myrepo
Public repositories can be cloned via HTTPS by anyone. Private repositories require SSH access from the tailnet.

Connection Status

# Check connectivity to git server
gitraf status

# Shows:
#   - Tailnet connection status
#   - SSH connectivity
#   - HTTPS connectivity
#   - Server version

GitHub Mirror Sync

Automatically mirror your repositories to GitHub for backup or collaboration.

Enable Mirroring

# Enable mirroring for a repository
gitraf mirror enable myrepo git@github.com:username/myrepo.git

# This creates a git-mirror.json config in the repo

Disable Mirroring

# Stop mirroring a repository
gitraf mirror disable myrepo

List Mirrors

# Show all repositories with mirroring enabled
gitraf mirror list

# Output:
#   myrepo        -> github.com:user/myrepo.git
#   another-repo  -> github.com:user/another.git

Manual Sync

# Sync a specific repository now
gitraf mirror sync myrepo

# Sync all mirrored repositories
gitraf mirror sync

Automatic Sync (Cronjob)

# Enable automatic sync via cron
gitraf mirror cron enable

# Choose schedule:
#   1. Hourly
#   2. Every 6 hours
#   3. Daily (midnight)
#   4. Custom cron expression

# Disable automatic sync
gitraf mirror cron disable

# Check cron status
gitraf mirror status
For GitHub mirroring to work, your server needs SSH access to GitHub. Add your server's SSH key to your GitHub account.

Pages Hosting

Deploy static sites from git repositories.

Enable Pages

# Enable pages for a repository
gitraf pages enable mysite

# Interactive prompts:
#   Branch: main
#   Build command: npm run build (optional)
#   Output directory: public

# Enable with options
gitraf pages enable mysite --branch main --output dist

Manage Pages

# List all pages-enabled repositories
gitraf pages list

# Check pages status and build info
gitraf pages status mysite

# Force rebuild and redeploy
gitraf pages deploy mysite

# Disable pages
gitraf pages disable mysite

Examples

Complete Workflow: New Project

# Create repository
gitraf create my-new-project --description "My awesome app"

# Clone locally
gitraf clone my-new-project
cd my-new-project

# Add files and push
echo "# My Project" > README.md
git add .
git commit -m "Initial commit"
git push origin main

# Make it public
gitraf public my-new-project

# Enable pages
gitraf pages enable my-new-project --output public

# Set up GitHub mirror
gitraf mirror enable my-new-project git@github.com:user/my-new-project.git
gitraf mirror cron enable

Migrate Existing Repository

# Create repo on gitraf server
gitraf create existing-project

# In your existing local repo
git remote add gitraf git@yourserver.ts.net:existing-project.git
git push gitraf main

# Or replace origin entirely
git remote set-url origin git@yourserver.ts.net:existing-project.git

Server Setup

Manual Installation

If you prefer manual setup over the deployment script:

1. Install Dependencies

apt update
apt install -y git nginx certbot python3-certbot-nginx

2. Install Tailscale

curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

3. Create Git User

useradd -m -s /bin/bash git
mkdir -p /opt/ogit/data/repos
chown -R git:git /opt/ogit

4. Configure SSH

Edit /etc/ssh/sshd_config to allow tailnet IPs:

Match Address 100.64.0.0/10
    PasswordAuthentication no
    PubkeyAuthentication yes

Nginx Configuration

For hybrid access (public HTTPS + tailnet SSH):

server {
    listen 443 ssl;
    server_name git.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/git.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/git.yourdomain.com/privkey.pem;

    # Block push from non-tailnet IPs
    location ~ ^/[^/]+\.git(/.*)?$ {
        if ($request_uri ~ /git-receive-pack$) {
            set $deny_push 1;
        }
        if ($remote_addr ~ ^100\.(6[4-9]|[7-9][0-9]|1[01][0-9]|12[0-7])\.) {
            set $deny_push 0;
        }

        proxy_pass http://127.0.0.1:8080;
    }

    location / {
        proxy_pass http://127.0.0.1:8081;
    }
}

SSL Certificates

# Standard certificate
certbot --nginx -d git.yourdomain.com

# Wildcard for pages (requires DNS challenge)
certbot certonly --manual --preferred-challenges dns \
    -d '*.yourdomain.com' -d yourdomain.com

Pages Hosting

Deploy static sites from git repositories to {repo}.yourdomain.com.

Enable Pages

gitraf pages enable mysite

You'll be prompted for:

  • Branch: Which branch to deploy from (default: main)
  • Build command: Optional command like npm run build
  • Output directory: Where the built site is (default: public)

Deployment Flow

  1. Push to the configured branch
  2. Post-receive hook runs your build command (if configured)
  3. Contents of output directory are deployed to {repo}.yourdomain.com

Example: Static Site

# Create repo
gitraf create my-blog
gitraf pages enable my-blog
# Branch: main, Build: (empty), Output: public

# Push your static files
git clone git@server:my-blog.git
cd my-blog
mkdir public
echo "<h1>Hello World</h1>" > public/index.html
git add . && git commit -m "Initial site"
git push origin main

# Site is now live at https://my-blog.yourdomain.com

Example: Build with npm

gitraf pages enable my-vite-app
# Branch: main
# Build: npm run build
# Output: dist

# Push triggers: npm install -> npm run build -> deploy dist/
Pages require a wildcard SSL certificate for *.yourdomain.com.

Git LFS

Store large files with S3-compatible storage (AWS S3, Cloudflare R2, etc.).

Setup

gitraf lfs setup

You'll be prompted for:

  • S3 endpoint URL
  • Bucket name
  • Access key ID
  • Secret access key

Usage

In your repository, track large files:

# Install git-lfs locally
apt install git-lfs  # or brew install git-lfs

# Initialize LFS in repo
git lfs install

# Track file types
git lfs track "*.psd"
git lfs track "*.zip"
git lfs track "videos/*"

# Commit .gitattributes
git add .gitattributes
git commit -m "Track large files with LFS"

File Size Limits

Files over 10MB are rejected unless tracked by LFS. This prevents accidentally committing large files.

S3 Provider Examples

Provider Endpoint
AWS S3 https://s3.amazonaws.com
Cloudflare R2 https://<account-id>.r2.cloudflarestorage.com
Backblaze B2 https://s3.<region>.backblazeb2.com
MinIO http://localhost:9000

Troubleshooting

Connection Issues

Can't connect via SSH

  • Verify Tailscale is running: tailscale status
  • Check server is reachable: ping yourserver.tail12345.ts.net
  • Verify SSH key is added: ssh-add -l

HTTPS returns 403

  • Repository might be private - connect via tailnet
  • Trying to push via HTTPS - only SSH supports push

Pages Issues

Site not updating after push

  • Check build command output: cat /opt/ogit/pages/myrepo/build.log
  • Verify output directory exists and contains files
  • Force re-deploy: gitraf pages deploy myrepo

Build fails

  • Ensure Node.js is installed for npm builds
  • Check if package.json exists
  • Verify build command is correct

LFS Issues

Push rejected: file too large

  • Track the file type with LFS: git lfs track "*.zip"
  • Re-add the file: git rm --cached file.zip && git add file.zip

LFS objects not downloading

  • Verify LFS is installed: git lfs version
  • Check S3 credentials in server config
  • Pull LFS objects: git lfs pull

General

Check server logs

# gitraf-server logs
journalctl -u gitraf-server -f

# nginx logs
tail -f /var/log/nginx/error.log

Restart services

systemctl restart gitraf-server
systemctl restart nginx
